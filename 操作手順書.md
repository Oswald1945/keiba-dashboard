# 競馬予想ダッシュボード 操作手順書

作業フォルダ: `C:\Users\r-ito\keiba-dashboard\`

---

## 全体フロー

```
【初回のみ】パッケージインストール
          ↓
① TARGET からCSV出力（全レース分）
          ↓
② Git Bash で python run_new.py → SmartRC自動取得 → 予想ダッシュボード HTML 生成・ブラウザで自動表示
```

> **SmartRC の自動取得について**
> `run_new.py` 実行時に SmartRC JSON が存在しない場合、`races/view` エンドポイントへの自動取得を試みます。
> 成功すれば STEP ② は不要です（DevTools での rcode 取得が不要になります）。
> 失敗した場合は従来通り `python smartrc_fetch.py --rcode XXXX --out` で手動取得してください。

---

## 【参考】SmartRC 手動取得が必要な場合（STEP ②）

自動取得が失敗した場合のみ実行します。

## 【初回のみ】必要パッケージのインストール

Python ライブラリが未インストールの場合、以下を **一度だけ** 実行してください。

**Git Bash** を開いて以下を実行:

```bash
cd /c/Users/r-ito/keiba-dashboard
pip install -r requirements.txt
```

> `pip` が見つからない場合は `python -m pip install -r requirements.txt` を使用してください。

インストールされるパッケージ:

| パッケージ | 用途 |
|-----------|------|
| pandas | CSVデータ処理 |
| numpy | 数値計算 |
| openpyxl | Excel (.xlsx) 読み込み |
| pykakasi | レース名ローマ字変換 |
| requests | SmartRC API通信 |

> 一度インストールすれば次回以降は不要です。

## STEP ①｜TARGET からCSVを出力する（全レース分まとめて）

当日予想したい全レース分を、以下の4種類まとめて出力します。

### ① - 1. TARGET を起動してレースを開く

1. TARGET を起動する
2. 「出馬表」タブで対象日・開催場・レースを選択する

---

### ① - 2. 出馬表を出力する

1. メニュー「ファイル」→「出力」→「**出馬表 Excel出力**」
2. ファイル名を以下の形式で入力して `input\` フォルダに保存:

```

例: 出馬表_20260601_tk11.xlsx
```

---

1. 出馬表画面で **Ctrl+A**（全馬選択）
2. メニュー「ファイル」→「出力」→「**過去成績 CSV出力**」
3. ファイル名を以下の形式で入力して `input\` フォルダに保存:

```
過去走_YYYYMMDD_場略称レース番号.csv
例: 過去走_20260601_tk11.csv
```

2. 「ファイル」→「出力」→「**坂路 CSV出力**」
3. ファイル名を以下の形式で入力して `input\` フォルダに保存:

```
坂路_YYYYMMDD_場略称レース番号.csv
例: 坂路_20260601_tk11.csv
```

### ① - 5. ウッドチップ調教データを出力する

```
ウッド_YYYYMMDD_場略称レース番号.csv
例: ウッド_20260601_tk11.csv
```

## STEP ②｜SmartRC から rcode を取得する（全レース分まとめて）

予想したい全レース分、以下の手順を繰り返します。

### ② - 1. SmartRC でレースを開く

1. Chrome で https://www.smartrc.jp/v3/ を開く
2. 画面左のカレンダーで対象日をクリック
3. 表示されたレース一覧から対象レースをクリック
4. 出馬表と「評価」列（A〜E）が表示されることを確認する

---

### ② - 2. DevTools で rcode を確認する

#### 1. DevTools を開く

**F12** を押す → 画面右または下に DevTools パネルが開く

#### 2. Network タブを開く

DevTools 上部のタブから「**Network**」をクリック

```
[ Elements ][ Console ][ Sources ][ Network ][ Performance ]...
                                      ↑ここ
```

#### 3. ページを再読み込みする

**F5** を押す（SmartRC のページが再読み込みされ、Network タブにリクエストが流れる）

#### 4. フィルターで絞り込む

Network タブ上部の検索ボックスに **`runners`** と入力する

```
[ 🔍 runners ]  ← ここに入力
```

→ 一覧に `runners/view?_dc=...` という行が表示される

> 表示されない場合は SmartRC 画面でレースをもう一度クリックする

#### 5. runners/view をクリックする

一覧の **`runners/view?_dc=...`** という行をクリック → 右側に詳細パネルが開く

#### 6. Payload タブを開く

詳細パネル上部のタブから「**Payload**」をクリック

```
[ Headers ][ Payload ][ Preview ][ Response ]...
              ↑ここ
```

#### 7. rcode をコピーする

以下のような内容が表示される:

```json
{
  "rcode": "2026060105010811",
  "toku": "0",
  "page": 1,
  "start": 0,
  "limit": 25
}
```

`"rcode":` の右の **16桁の数字**（例: `2026060105010811`）をドラッグして選択し **Ctrl+C** でコピー

> ダブルクォーテーション `"` は不要。数字だけコピーしてください。

---

### ② - 3. Git Bash でデータを取得・保存する

**Git Bash** を開き、以下を実行する（`--rcode` の後にコピーした16桁を貼り付け）:

```bash
cd /c/Users/r-ito/keiba-dashboard
python smartrc_fetch.py --rcode 2026060105010811 --out
```

> Git Bash での貼り付けは **Shift+Insert**

成功すると以下のように表示される:

```
[OK] セッション取得完了
[INFO] rcode=2026060105010811  ->  race_id=20260601_tk11
[INFO] runners/view 取得中...
  -> 16 頭取得
[保存完了] C:\Users\r-ito\keiba-dashboard\smartrc_20260601_tk11.json
```

`keiba-dashboard\` 直下に **`smartrc_20260601_tk11.json`** が自動生成される。  
（rcode から日付・場・レース番号が自動解析されるのでリネーム不要）

②-1〜②-3 を全レース分繰り返す。

## STEP ③｜予想ダッシュボードを生成する

全ファイルの準備ができたら、Git Bash で1コマンド実行するだけです。

```bash
cd /c/Users/r-ito/keiba-dashboard
python run_new.py
```

**自動で行われること:**

1. `input\` のファイルをレースIDでグループ化
2. `smartrc_{race_id}.json` を自動検出して SmartRC 評価補正（過去5走加重平均 最大±4.5pt）を適用
3. スコア計算・HTML 生成
4. 完了ファイルを `input\done\` へ移動

**実行ログの見方:**

```
[Race] 20260601_tk11
  検出: 過去走, 出馬表, 坂路, ウッド
  [SmartRC] 評価補正を適用: smartrc_20260601_tk11.json
  [SmartRC] 16頭分のデータを読み込みました
    [SmartRC] アーバンシック: 評価=A (+4.50pt)  推定人気=2(3位)
    [SmartRC] レガレイラ:     評価=C (+0.00pt)  推定人気=1(1位)
  ...
■ ペース: スローペース想定
  1位予想: アーバンシック  score=68.3  (差し)
  [OK] 20260601_tk11 完了 -> done/ へ移動

[Race] 20260601_hs8
  ...
=== 完了 ===
```

> **SmartRC評価補正の仕組み:** 過去5走分の馬場・展開有利不利評価（A〜E）を加重平均（前走×1.0・前々走×0.6・3走前×0.3・4走前×0.15・5走前×0.07）でスコア化。A=+4.5pt / B=+2.5pt（不利受けた→上方修正）、D=−2.5pt / E=−4.5pt（有利に恵まれた→下方修正）、C=0pt。

**生成されるファイル:**

```
C:\Users\r-ito\keiba-dashboard\
├── pred_20260601_tk11.html    ← ブラウザで自動表示される
└── pred_20260601_hs8.html
```

スクリプト完了後、生成された HTML ファイルが **ブラウザに自動で表示**されます。

### ③ - オプション｜再生成・強制上書き

予想 HTML を生成済みでも、スコアロジックやデータを更新した後に**再生成したい場合**は `--force` を付けます:

```bash
python run_new.py --force
```

> `--force` なしだと `horses_data_{race_id}.json` が存在する限りスキップされます。

## トラブルシュート

| 症状 | 原因 | 対処 |
|------|------|------|
| `[pred] 生成済 -> スキップ` | JSON が残っている | `--force` オプションを付けて再実行 |
| SmartRC評価がすべて 0.0pt | JSON が古い (h_fr_baba形式) | `smartrc_fetch.py --rcode ... --out` で再取得 |
| `[保存完了]` が出るのにJSONが壊れる | 書き込み中断 | ファイルサイズを確認（正常なら 30KB 以上） |
| `races/view` が全パターン失敗 | SmartRC の API 仕様変更 | `--rcode` で直接指定（DevTools で取得） |

## 効率化のヒント

### 現在の手順の手間

| ステップ | 手間 | 課題 |
|---------|------|------|
| STEP ① TARGET CSV出力 | レースごとに4ファイル × Nレース | ファイル命名を正確に行う必要がある |
| STEP ② SmartRC rcode取得 | DevTools を開いて rcode を手動コピー × Nレース | 最も手間のかかる工程 |
| STEP ③ run_new.py | 1コマンドで完結 | 問題なし |

### 将来的な効率化案

**① SmartRC自動取得の組み込み（最大の改善）**

`run_new.py` 実行時に SmartRC JSON がなければ自動取得を試みる機能。`races/view` エンドポイントが成功すれば STEP ② が完全に不要になります。レースIDから日付・場コードが判明しているため技術的には実装可能です。

**② --date モードで当日全レース一括取得**

SmartRC の `races/view` が使える日は以下で当日の rcode 一覧を確認できます:

```bash
python smartrc_fetch.py --date 20260601
```

一覧が取得できれば、場コードとレース番号を指定して rcode を自動解決できます（`python smartrc_fetch.py 20260601_tk11 --out`）。

**③ rcodeを覚えておく**

同じレースを何度も再取得する場合は rcode をメモしておくと便利です。rcode は16桁固定で同一レースは変わりません。

> 自動表示されない場合は、エクスプローラーで `keiba-dashboard\` を開き、`pred_YYYYMMDD_場略称R番.html` をダブルクリックして確認してください。

## レース結果入力後（回顧生成）

レース後に結果 CSV を `input\` へ追加し、`python run_new.py` を再実行すると回顧ダッシュボードが生成される。

```
input\ に追加するファイル:
レース結果_20260601_tk11.csv
```

## トラブルシューティング

| 症状 | 原因 | 対処 |
|------|------|------|
| `過去走 or 出馬表がない -> スキップ` | ファイル名の形式が違う | `過去走_YYYYMMDD_場略称R番.csv` の形式か確認 |
| SmartRC 補正が適用されない | JSON の場所が違う | `keiba-dashboard\` 直下（`input\` の外）に置く |
| SmartRC 補正が適用されない | ファイル名の race_id が不一致 | JSON ファイル名が `smartrc_過去走と同じ名前.json` になっているか確認 |
| `[ERROR] runners/view エラー` | SmartRC セッション切れ | Chrome で SmartRC を開き直し、F5 で再読み込みしてから rcode を再取得 |
| `runners/view` がフィルターに出ない | Network タブを開く前にページを読み込んだ | F5 で再読み込みする |
| Git Bash で `python` が見つからない | PATH の問題 | Git Bash を閉じて再起動してから試す |
